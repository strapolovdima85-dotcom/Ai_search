# Структура проекта: Интеллектуальный поиск номенклатуры в 1С:ERP

В данном документе описано назначение каталогов и ключевых файлов архитектуры ИИ-компонента умного поиска.

## Иерархия каталогов

```text
Ai_search/
├── 1c_components/             # Компоненты на стороне 1С:ERP
│   └── vneshnyaya_obrabotka.txt # Код внешней обработки/расширения для 1С (HTTP-запросы)
│
├── ai_service/                # Микросервис поиска на Python (FastAPI)
│   ├── app/                   # Исходный код приложения
│   │   ├── api/               # Эндпоинты и маршрутизация API
│   │   │   └── endpoints.py   # Обработка маршрутов (например, /search, /update)
│   │   ├── core/              # Конфигурация и инициализация
│   │   │   └── config.py      # Настройки приложения, путей и доступов к БД
│   │   └── services/          # Бизнес-логика ИИ
│   │       ├── search_engine.py # Логика гибридного поиска в Векторной БД
│   │       └── embedder.py    # Класс генерации эмбеддингов через HuggingFace/PyTorch
│   │
│   ├── models/                # Локальное хранилище весов ИИ-модели (сache/дообученная)
│   ├── requirements.txt       # Зависимости проекта (fastapi, qdrant-client, transformers)
│   └── Dockerfile             # Инструкция для контейнеризации и деплоя сервиса
│
├── data/                      # Управление данными номенклатуры
│   ├── raw/                   # Сырые выгрузки номенклатуры из 1С (JSON/CSV)
│   └── processed/             # Очищенные данные, готовые для векторизации
│
├── notebooks/                 # Jupyter-ноутбуки для R&D и экспериментов
│   ├── 01_data_preprocessing.ipynb # Анализ данных 1С, очистка дублей и токенизация
│   └── 02_model_training.ipynb    # Дообучение/валидация эмбеддинг-модели (SentenceTransformers)
│
├── docs/                      # Проектная и дипломная документация
│   ├── manifest.md            # Бизнес-ценность и манифест проекта
│   └── architecture.md        # Техническая архитектура и Data Flow системы
│
├── STRUCTURE.md               # Данное описание структуры (карта проекта)
└── README.md                  # Главная инструкция по запуску и развертыванию
```

## Назначение ключевых папок для дипломной работы

1. **`1c_components/`** — подтверждает интеграционную часть диплома. Здесь хранится код на языке 1С, который отвечает за отправку JSON-запросов и парсинг ответов от ИИ.
2. **`ai_service/app/services/`** — "сердце" вашей ИИ-архитектуры как Архитектора ИИ. Показывает комиссии, как именно вы управляете ML-моделями и векторным поиском.
3. **`notebooks/`** — научно-исследовательский раздел. Ноутбуки служат доказательством проведения экспериментов, тюнинга гиперпараметров и обоснования выбора конкретной NLP-модели.
4. **`data/`** — демонстрирует работу с реальным датасетом организации, соблюдение принципов гигиены данных и разделение процессов на Offline-обработку и Online-поиск.
